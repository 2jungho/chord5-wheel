# 안드로이드 모바일 웹 디버깅 가이드

안드로이드 기기에서 Chrome 브라우저로 웹앱을 실행할 때 발생하는 문제를 PC에서 디버깅하는 방법입니다.

## 방법 1: 화면 내 디버거 사용 (vConsole) **(추천)**

현재 `index.html`에 **vConsole** 도구를 추가해 두었습니다. 앱을 다시 배포하고 실행하면 화면 우측 하단에 녹색 **vConsole** 버튼이 나타납니다.

1.  웹앱 접속 후 우측 하단 녹색 버튼 터치.
2.  **Log** 탭에서 붉은색 에러 메시지를 확인.
3.  **Network** 탭에서 로딩 실패한 파일(빨간색)이 있는지 확인.

## 방법 2: PC와 USB 연결하여 디버깅 (Chrome Remote Debugging)

PC의 크롬 브라우저 개발자 도구를 이용해 모바일 화면을 그대로 검사할 수 있는 정석적인 방법입니다.

### 1단계: 안드로이드 기기 설정
1.  **설정 > 휴대전화 정보 > 소프트웨어 정보**로 이동.
2.  **빌드 번호**를 7번 연속 터치하여 '개발자 모드' 활성화.
3.  **설정 > 개발자 옵션**으로 이동하여 **USB 디버깅**을 켭니다.

### 2단계: PC와 연결
1.  USB 케이블로 PC와 안드로이드를 연결합니다.
2.  휴대전화 화면에 "이 컴퓨터의 USB 디버깅을 허용하시겠습니까?" 팝업이 뜨면 **허용**을 누릅니다.

### 3단계: PC Chrome에서 디버깅
1.  PC에서 Chrome 브라우저를 켭니다.
2.  주소창에 `chrome://inspect/#devices` 를 입력하고 엔터를 칩니다.
3.  **Devices** 목록에 자신의 안드로이드 기기 모델명이 보여야 합니다.
4.  안드로이드 기기에서 디버깅하려는 웹사이트(`https://chord5-wheel.web.app`)를 엽니다.
5.  PC 화면 목록에 해당 탭이 나타나면 아래의 **Inspect** 버튼을 클릭합니다.
6.  새로운 창이 뜨며 모바일 화면과 개발자 도구(Console, Network 등)를 PC에서 볼 수 있습니다.

---

## 주요 확인 포인트

빈 화면이 나올 때 **Console** 탭에서 다음 에러들을 찾아보세요:

*   **HTML renderer error**: 렌더러 초기화 실패.
*   **Failed to load resource**: `main.dart.js`, `Tone.js` 등의 파일 로딩 실패 (404 등).
*   **CORS policy / COOP / COEP**: 보안 정책으로 인한 차단.
*   **Uncaught SyntaxError**: 자바스크립트 문법 오류 (구형 기기 호환성).
